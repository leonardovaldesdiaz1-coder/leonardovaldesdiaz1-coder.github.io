1. Tecnologías de Web APIs

Definición de API

Una API permite que diferentes aplicaciones se comuniquen entre sí mediante reglas y protocolos definidos.

SOAP

Protocolo basado en XML usado en entornos empresariales con alta seguridad.

  • Mensajes estructurados en XML
  • Estándares formales
  • Mayor complejidad

API REST

Estilo arquitectónico ligero que utiliza HTTP.

  • GET, POST, PUT, DELETE
  • Uso común de JSON
  • Flexible y escalable

2. Arquitectura de Microservicios

Modelo donde una aplicación se divide en pequeños servicios independientes.

  • Despliegue y escalado independiente
  • Comunicación vía REST o eventos
  • Mayor agilidad empresarial

Diferencia con monolito: El monolito es una sola aplicación grande y acoplada.

3. Desarrollo de una Web API

Recomendaciones
  1. Definir objetivos
  2. Seguir estándares REST
  3. Documentar con Swagger o Postman
  4. Priorizar seguridad
  5. Versionar la API
Proceso
  1. Análisis de requerimientos
  2. Planificación
  3. Diseño de endpoints
  4. Implementación
  5. Pruebas
  6. Despliegue
  7. Monitoreo y mantenimiento
Ejemplo de Endpoints
  • /posts
  • /posts/{id}
  • /comments
  • /comments/{id}

4. Contenedores

Los contenedores son paquetes ligeros que incluyen el código y dependencias necesarias para ejecutar aplicaciones en cualquier entorno.

  • Virtualizan a nivel de sistema operativo
  • Comparten el kernel
  • Son más ligeros que máquinas virtuales
  • Permiten portabilidad y escalabilidad

Uso: Empaquetar aplicaciones y ejecutarlas de forma uniforme en nube o servidores.
